Wallet User Guide

A multi -coin multi-platform tip, soak, utility bot

Getting started

To get started with Wallet you need to ensure that you have registered and identified with NickServ on Freenode. You can register using the following command:

/msg NickServ REGISTER <password> <email-address>

Once registered and identified with NickServ you are ready to go, there's no need to create an account with Wallet. You can increase your account balance in two ways; receiving coins through tips, soaks and rain or through depositing directly from a wallet.

Coins

To view all the coins that Wallet current supports use the following command:

/msg Wallet coins

Idle time (bonus time)

Wallet uses the concept of idle time which allows a user to still be included in soaks for up to 24 hours without needing to talk in the channel every 10 minutes to remain active. Whilst a user will still remain active for 10 minutes after talking it is possible to earn much greater amounts of idle time by soaking and raining. Idle time also allows you to remain active even if you are offline, meaning that you will still be included in soaks even if you disconnect from IRC completely. This significantly cuts down on the amount of spam generated in the channel whilst also rewards those who give back to the channel by soaking and raining.

Soak

A soak allows a user to choose a specific amount of coins to be distributed evenly among the active users within a channel. Soaks can either be public where the channel can see who soaked the coins or ninja where the soaker is not announced to the channel. Soaking will soak active and bonustime users.

Example: When there are 10 active users in a channel and a user decides to soak 1000 coins each active user will receive 100 coins each.

To public soak use the command: (<minutes> is optional)

.soak <amount> <minutes>
.soak ltc <amount> <minutes>
.soak xzc <amount> <minutes>

To ninja soak use the following command to send a message directly to the bot:

/msg Wallet soak <amount> <minutes>
/msg Wallet soak ltc <amount> <minutes>
/msg Wallet soak xzc <amount> <minutes>

To soaknow use the following commands: (soaknow does NOT include BonusTime users)

.soaknow <amount>

Soak walkthrough

This walkthrough will show and explain what happens when a soak occurs in the channel. Often times new users are confused about what the response from Wallet means after a soak so this walkthrough aims to breakdown the response and explain each part thoroughly.

A user soaks 10 DOGE in the channel as so.

.soak 10

Wallet then outputs something that looks like the following:

<+Wallet> user1 soaked 0.2 DOGE each: 50 users (See prev list or whogot) |
.whogot <alphanumeric> | Idle: 6 -> 15 min | @BonusPot + 0 = 456.56837088

Breakdown of the response

Wallet will first split the soak evenly between all active users, 50 in this case so each user will receive 0.2 coins each. 

<+Wallet> user1 in #channel soaked 50 with 0.2 DOGE each

Scraps & BonusPot

Any left over coins that cannot be divided as whole coins will go to the bot as scraps, 0 coins in this case. Scraps go to the Wallet channel balance and are used to fund bonus soaks whilst also helping to pay for server costs which keep the bot running. 

@BonusPot + 0 = 456.56837088

Idle bonus

The idle bonus is a random multiplier that assigns you an amount of idle time that roughly correlates to the amount of coins that you soaked, depending on luck you may receive more or less than what you soaked. In this case the user received 9 minutes of idle time for their soak which when added onto their already banked time brought them up to 15 minutes total.

Idle: 6 -> 15 min

Soaked users

The last part of the syntax is the users who are included in the soak, these are users who are active in the channel either by talking within the last 10 minutes or by having idle time. This allows users to be notified when they are included in soaks.

user2 user3 user4 user5 user6 user7 user8 user9 user10 user11 user12 user13

If the list is to large the bot will give a whogot code <alphanumeric> to display the users. 

.whogot <alphanumeric>

Reserve

Reserves are secondary balances separate from the one used for tipping and soaking.

Balances

/msg Wallet balance reserve
/msg Wallet ltc balance reserve
/msg Wallet xzc balance reserve

Adding to Reserve

/msg Wallet reserve <amount>
/msg Wallet reserve ltc <amount>
/msg Wallet reserve xzc <amount>

Releasing from Reserve

/msg Wallet release <amount>
/msg Wallet release ltc <amount>
/msg Wallet release xzc <amount>

2 Factor Authentication (2FA)

2FA is an OPTIONAL piece of security afforded to the users.

/msg Wallet secret <action>

Actions: set clear

  • Set: Bot PM's your 16 char TOTP 2FA (ie Google Authenticator) key.
  • Clear: N/A - WARNING: No confirmation process. Enabled IMMEDIATELY after 'set'.

Unlocking 2FA

To unlock 2FA to release coins from your reserve you must use the following command: (Your reserves will remain open for 120 seconds.)

/msg Wallet unlock <2FA code (6 digit)>

Jackpot

The jackpot command allows users a chance of triggering one of the following actions:

If tipped LESS than 50:

  • Random tip returned to user or to another random user.

If tipped MORE than 50:

  • The tipped amount is returned to the user OR tipped to a different user in the channel. (Bonus multiplier 1x-3x)

If tipped MORE than 50 AND BonusPot balance is GREATER than "Approximate Target":
"Approximate Target" = Pseudo random number

  • The BonusPot soaks a random amount to the channel at a random time period.
  • The BonusPot rewards the top10 list a random amount at a random time period.

To contribute to the jackpot and be in with a chance of triggering the jackpot use the following command:

.jackpot <amount>
